
## Intelligent Diagnosis of Inter-Turn Short Circuit Faults in Stator Windings of Three-Phase Permanent Magnet Synchronous Motors Based on 1D Convolutional Neural Network

This code repository is the open-source code accompanying an unpublished paper, authored by Haoxiang Xu.

## Guide
This project only provides the benchmark accuracy of closed-set tests and cross-operating condition tests conducted on the self-developed laboratory dataset, as well as a standardized intelligent fault diagnosis code repository. This code repository enables users to independently construct diagnostic test tasks and optimize diagnostic network architectures based on this dataset, thereby further carrying out innovative research work such as transfer learning, domain generalization, and fault severity identification. All experiments were conducted on the Windows 10 operating system with the PyTorch 2.3.1 deep learning framework; the hardware configuration for the experiments included an Intel Core i7-12700KF processor, an NVIDIA GeForce RTX 4060 Ti graphics card, and 16 GB of RAM.

## Datasets
A multi-operating-condition combined test scheme was adopted for dataset collection, focusing on Inter-Turn Short Circuit (ITSC) faults in the stator windings of Permanent Magnet Synchronous Motors (PMSMs). The test conditions covered 3 speed levels (10%, 20%, and 30% of the rated speed) and 4 torque levels (0.07%, 6.83%, 20.49%, and 34.14% of the rated torque). For each speed-torque operating condition, ITSC fault conditions with different percentages of short-circuited turns (¦Ì) and different short-circuit resistances (Rf) were sequentially applied to the three-phase windings (Phase A, B, and C) of the prototype motor, aiming to systematically investigate the variation patterns of fault characteristics with operating conditions and fault parameters.

For the percentage of short-circuited turns in the stator windings of the prototype, a total of 9 gradient levels were set, covering the range from 1.98% (5/252) to 45.63% (115/252). The specific levels are as follows: 1.98%, 3.97%, 5.95%, 8.33%, 12.30%, 20.64%, 28.97%, 37.30%, and 45.63%. To simulate the dynamic variation characteristics of fault loop impedance during insulation degradation in practical engineering, multiple levels of short-circuit resistance (Rf) were introduced in the experiment. Without an external resistor connected, the total resistance of the loop composed only of external power test wires is approximately 10 m¦¸ (i.e., the minimum short-circuit resistance Rf in the experiment is 0.01 ¦¸). By replacing the external resistor, Rf was further set to two levels (1 ¦¸ and 10 ¦¸) to restore the impedance characteristics at different fault development stages.

Based on the above experimental design, a total of 972 groups of valid data (including the healthy state and various fault states of the motor) were collected in this study, with a total data volume of approximately 128 GB. All data were synchronously collected at a sampling frequency of 1 MHz and a bandwidth of 50 kHz, with a recording duration of 5 seconds per group of data. The collected signals include three-phase voltage, three-phase current, and short-circuit current, which completely retain the time-domain information of fault characteristics.

All data are stored in the .MAT (MATLAB) format, a commonly used format in the scientific research field, and the naming of all data files follows unified coding rules, allowing the corresponding experimental conditions to be directly parsed from the file names. The file naming format is: [Rotation Frequency]-[Torque]-[Short-Circuit Resistance]-[Fault Phase]-[Percentage of Short-Circuited Turns]-[Control Mode].MAT. Taking the file name "20-0.07-0.01-A-1.98-FOC" as an example, it indicates that the motor operates at a frequency of 20 Hz, the load torque is 0.07% of the rated torque, the external short-circuit resistance is 0.01 ¦¸, the fault phase is Phase A, the percentage of short-circuited turns is 1.98%, and the motor adopts the Field-Oriented Control (FOC) strategy.

Each .MAT file contains multiple variables, among which "WaveInfo" is auxiliary information generated by the wave recorder, which has no practical analytical value and can be ignored. Valid experimental data are stored in 7 variables named "CH0x", each of which is a one-dimensional single-precision array with a length of 5¡Á10? (corresponding to a 5-second collection duration). The correspondence between variables and physical signals is as follows: CH01 corresponds to Phase A voltage, CH02 corresponds to Phase A current, CH03 corresponds to Phase B voltage, CH04 corresponds to Phase B current, CH05 corresponds to Phase C voltage, CH06 corresponds to Phase C current, and CH08 corresponds to short-circuit current.

## Pakages

The functional definitions of each directory and core file in this code repository are as follows:
- [checkpoint]. Stores log files generated by each model run, as well as record information of the optimal model parameters during the training phase.
- [ITSC_Datasets]. The dataset loader, which supports the configuration of diversified training tasks, including the dataset loading logic for scenarios such as cross-speed transfer performance verification and cross-torque transfer performance verification.
- [models]. Stores the relevant code for the 1D Convolutional Neural Network (1D-CNN) model adopted in this study.
- [utils]. Encapsulates various utility functions required to implement the entire model training process.
- [Input_data_per.py]. The pre-collection program for network input data, responsible for the pre-collection of model input data.
- [train.py]. The main program for model training, whose core functions include parsing training parameters, configuring the training environment, initializing training tool classes, and executing the complete model training process.

## Usage
- Download the dataset. 
- Determine whether it is necessary to pre-collect the input data required for the cross-speed operating condition test task. If required, run the Input_data_per.py file. Please read the comments in this code in advance before use. 
- Adjust the default parameters in the argument parser (parser) within the train.py file according to the diagnostic task. The comments in this code have provided detailed explanations of the parameters to be adjusted and the corresponding operation procedures.
- Other parameters can be adjusted independently according to the user's specific requirements for their diagnostic task. Each .py file contains detailed descriptions of the usage procedures and precautions.
- Note: If no adjustments are made to the parser parameters, the code will execute the diagnostic task of "a single migration test from the 0.07% rated torque operating condition to the 34.14% rated torque operating condition at 20 Hz".
